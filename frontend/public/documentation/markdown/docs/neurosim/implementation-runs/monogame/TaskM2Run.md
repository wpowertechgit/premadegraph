
**VegetationRenderer.PlaceTierSettlement() — tier-aware placement:**
1. Resolves `SettlementVisualProfile` via `AssetRegistry.ResolveSettlement(tier, biome)`
2. If model is loaded, places it via `AddStaticInstance` with scale fitted to tile
3. If model NOT loaded, walks tier fallback chain: Empire→Kingdom→Duchy→City→Tribe
4. Ultimate fallback: Kenney compound (always available)

**GameRoot.LoadVegetationModels() — loads all 7 FBX models using catalog keys:**
```csharp
foreach (var asset in RuntimeAssetCatalog.SettlementModels)
    _vegetationRenderer.LoadModel(asset.Key, asset.RelativePath);
```
Models stored under catalog keys (`settlement/tribal_green` etc.), matching `SettlementVisualProfile.ModelAssetKey` values from AssetRegistry. This is critical — using file paths as keys would cause key mismatch with AssetRegistry lookups.

## Risks / Follow-Ups

1. **High-poly FBX models**: `tribal_green.fbx` is ~726k vertices. Current implementation places one model per tribe. With 12 tribes, this means 12 high-poly meshes. Monitor performance. M13/M17 address LOD.
2. **FBX texture resolution**: Materials/textures load via `TryLoadSiblingDiffuseTexture()`. If a model's textures aren't found as sibling PNGs, it renders with fallback diffuse color. Check `asset-load.log` after first run.
3. **Demo tier limitation**: All local demo tribes are `Tier = PolityTier.Tribe`. Higher-tier settlement models won't render until tribes merge upward. M18B addresses this with a stress preset.
4. **Scale policy**: Settlement models are scaled to fit 34 world units (`FitModelToTile(modelKey, 34f)`). May need tuning per model after visual review.
5. **Draw order**: Settlements render with vegetation (same `Render()` pass). No z-fighting observed in existing tribal_green rendering. Monitor as more models appear.

## Bugfix — Model Key Mismatch (2026-05-06)

**Symptom:** Settlements disappeared after runtime follow-up. No tribal_green model visible.

**Root cause:** `LoadVegetationModels()` used file paths as model keys (`"Models/Settlements/Tribal_Green/tribal_green"`), but `SettlementVisualProfile.ModelAssetKey` uses catalog keys (`"settlement/tribal_green"`). `PlaceTierSettlement` → `ResolveLoadedSettlementKey` → `_loadedModels.ContainsKey("settlement/tribal_green")` was always false because models stored under file-path keys.

**Fix:** Changed `LoadVegetationModels()` to iterate `RuntimeAssetCatalog.SettlementModels` and call `LoadModel(asset.Key, asset.RelativePath)`. Models now stored under catalog keys matching AssetRegistry lookups.
